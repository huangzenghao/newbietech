使用 Hexo 在 CODING 和 GitHub 上搭建博客

发表于 2018-01-10 | 分类于 笔记 ， IT | 评论数：

我的目标是：

  • 在 CODING 和 GitHub 上启用 Pages 服务；
  • 绑定自定义域名；
  • 强制 HTTPS 访问；
  • 在 CODING 上创建 hexo 分支，保存源文件，以便在不同电脑间使用时同步。

准备工作

域名解析

我提前将域名以 cname 的方式解析至 pages.coding.me ，以便绑定自定义域名的同时，可以成功申请 SSL/TLS 证书。

具体操作可参照域名解析服务商的文档。

添加部署密钥

由于我原先已经为 CODING 和 GitHub 添加过密钥，本次就不再生成，直接将密钥复制过来并修改权限：

如看到与上面类似的内容，表明已经正确设置了读写权限。

根据提示创建成功后，将公钥，即 id_rsa.pub 文件的内容粘贴到 CODING 和 GitHub 相应的位置上，即完成部署密钥的添加。

为方便使用，创建或复制密钥时不要使用 sudo 提权，否则进行部署操作时不使用 sudo 提权则会提示权限不足。

启用 Pages 服务

我启用了 CODING 和 GitHub 的 Pages 服务，部署来源均设置为 master 分支，当然也可以参照说明文档设置为其他分支。

绑定域名的同时，如果已经正确设置了域名解析，则会成功自动申请 SSL/TLS 证书，如域名解析设置不正确，或未生效，则需待正确设置并生效后，方可成功申请到证书。

强制 HTTPS 访问处我设置为开启。
